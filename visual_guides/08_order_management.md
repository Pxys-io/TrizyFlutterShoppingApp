# Order Management Visual Guide

## Overview
The order management system allows users to view their order history, track order status, and manage order-related activities using the store backend API.

## API Endpoints Used
- `GET /api/orders/get-user-orders` - Get user's orders
- `GET /api/orders/get-order-details/:orderId` - Get specific order details
- `GET /api/orders/get-latest-order-details` - Get latest order details
- `GET /api/payments/check-order-status` - Check order payment status

## Order Actions
### Available Actions Based on Status
- **Pending**: Cancel order
- **Processing**: Cancel order, Contact support
- **Shipped**: Track package, Contact support
- **Delivered**: Rate/Review products, Return request

## UI/UX Considerations
- Order status visualization
- Timeline view for order progress
- Action buttons based on status
- Order summary at a glance
- Sorting and filtering options
- Return/exchange options
- Support contact options

## State Management
- Order history provider
- Order detail provider
- Order status state
- Loading states for order data
- Order refresh functionality
